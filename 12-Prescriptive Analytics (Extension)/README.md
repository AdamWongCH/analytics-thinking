# 12. Prescriptive Analytics

---

## Learning Objectives

By the end of this module, you should be able to:

* understand what prescriptive analytics is
* move from prediction to decision
* prioritise actions under constraints
* apply simple optimisation thinking

---

## Why This Module Matters

So far, you have learned how to:

* understand data
* build models
* make predictions

But predictions alone do not create value.

They answer:

> *What is likely to happen?*

We still need to decide:

> **What should we do?**

---

## The Core Idea

Prescriptive analytics answers:

> **Given what we know, what is the best action?**

It completes the progression:

**understanding → prediction → decision → action**

---

## Scenario

You are given a list of students with risk scores.

You can only intervene on a limited number of students.

Your task is to decide:

* who should be prioritised
* how to allocate limited resources

---

## Framework

CRISP-DM Phase:

1. Business Understanding
2. Data Understanding
3. Data Preparation
4. Modelling
5. **Evaluation** ← comparing decision options
6. **Deployment** ← taking action

Prescriptive analytics focuses on:

* choosing the best action during evaluation
* implementing decisions during deployment
